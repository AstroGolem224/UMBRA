# gemini cli / UMBRA workbench bootstrap

## execution contract

du wurdest von UMBRA in einem konkreten workspace gestartet. halte dich an den aktuellen repo-kontext und bearbeite nur dateien in diesem workspace.

## expected output

1. schreibe klar, was du getan hast
2. liste betroffene dateien auf
3. nenne tests oder validierungen
4. markiere blocker klar

## workspace rules

1. keine dateien ausserhalb des aktuellen workspaces aendern
2. vor aenderungen erst lesen und planen
3. bei task-mode konkrete umsetzung bevorzugen, nicht nur analyse

## uap optional

wenn UAP env-vars vorhanden sind, kann der run-status zusaetzlich an UMBRA gemeldet werden.
